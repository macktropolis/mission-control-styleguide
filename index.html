Mission Control
Style Guide

Design tokens, component patterns, and usage rules for the Mission Control FileMaker Developer Portal. All UI decisions derive from the Dark Ops / Aerospace aesthetic — precision over decoration, signal over noise.

All Futura / Century Gothic display text is uppercase with generous letter-spacing — this is a core part of the aesthetic. Do not use the display font at sentence case for UI chrome. Menlo / Consolas remains in its natural mixed case for readability. Because Futura and Century Gothic have slightly different proportions, test heading sizes on both platforms when finalizing layout.

The base unit is 10px (one minor grid step), with 5px available as a half-step for fine adjustments. All object positions and sizes snap to this unit. The major grid at 50px governs structural layout — panel heights, row heights, and column widths should be multiples of 50px wherever possible. Use minor steps (10–40px) for internal padding, gaps, and fine alignment within objects; use 5px only for micro-adjustments where a full 10px step is too coarse.

04 Visual Effects
Scanline Overlay
Fixed full-screen overlay using repeating-linear-gradient. 2px transparent / 2px rgba(0,0,0,0.07). Applied via body::before. Adds CRT texture without obscuring content.
Background Grid
Subtle 40×40px grid on the main content area. Uses bg-card color on the bg color — barely visible, suggests a technical plotting surface.
Ambient Glow
Radial gradient in top-left corner via body::after. Breathes slowly (8s cycle). Blue dim opacity creates a light source impression — control room ambience.
Edge Light
Top-edge gradient line on hover: linear-gradient(90deg, var(--blue), transparent). Applied via ::before pseudo-element with opacity 0 → 1 on hover.
Gradient Borders
Sidebar uses a right-side vertical gradient line (top-to-bottom fade). Topbar uses a horizontal gradient line (left-to-right fade). Both use ::after pseudo-elements.
Crosshair Ornament
SVG radar/crosshair used as decorative element in the page header. Opacity 0.06–0.1. Should always be blue and positioned to the far right of its container.
05 Motion
fadeUp — Panel Entry
Panels animate in on load: opacity 0→1 + translateY 12px→0 over 0.4s ease. Staggered with 0.05s delay increments per panel. One-time reveal on load only.
breathe — Ambient Glow
Body ambient glow pulses scale 1→1.05 + opacity 0.4→0.7 over 8s ease-in-out infinite. Subtle — barely perceptible. Creates a "live system" quality.
pulse — Status Indicator
Status dots fade opacity 1→0.3 over 2s ease-in-out infinite. Used on the "Systems Nominal" pill in the topbar. Conveys a live/active state.
blink — Cursor
Terminal cursor after page titles: 1.2s step-end infinite. The cursor appears only on the primary dashboard heading. One per screen maximum.
→
Hover Transitions
All interactive elements use transition: all 0.15s. Arrow indicators translate from -4px to 0 on hover. Border colors transition from --border to --border-lit. Never use 0s transitions for UI elements.
06 Buttons
Variants

Primary — Solid Dodger Blue with black text. Use for the single most important action on a screen. Glows on hover. Ghost — Border-only. All secondary actions. Border transitions to blue on hover. Danger — Red-tinted ghost. Destructive actions only. Use sparingly.

Sizes

Default padding: 7px 14px. Small: 4px 10px. Icon-only: 7px 10px. All buttons use Space Mono, 10px, uppercase, 0.1em letter-spacing.

/* Base button */ .btn { font-family: var(--mono); font-size: 10px; letter-spacing: 0.1em; text-transform: uppercase; padding: 7px 14px; border-radius: 3px; border: 1px solid transparent; transition: all 0.15s; } .btn-primary { background: var(--blue); color: #000; font-weight: 700; } .btn-primary:hover { background: #4aaeff; box-shadow: 0 0 16px var(--blue-glow); }

08 Panels
◈ Panel Title
12 entries

Panels have three layers: card bg (#0f1520), header with 3% blue tint, and a bottom-border separator. Border transitions from --border to --border-lit on hover. The top edge-light activates on hover via ::before. Panel titles always use Chakra Petch, 11px, 600 weight, 0.18em spacing, uppercase. 12 Inputs

Inputs use bg-card background, --border default, 12px Space Mono. On focus: border transitions to --blue-dim and a 3px --blue-faint ring appears. Placeholder text uses --text-faint. Border-radius is 3px (same as buttons) for consistency.

13 Iconography
Icon Containers
◈
Small 24px
🔧
Medium 32px
⚡
Orb 36px

Three icon container shapes: square (list items and nav), square larger (panel accents), and circular orb (tool grid). All use --blue-faint fill and --blue-dim border. The orb adds a radial gradient for dimensionality and a subtle outer glow.

Icon Style

All icons are inline SVG at 1.5px stroke weight, no fill. Navigation icons are 14×14px. Inline panel-title icons are also 14×14px. In icon containers they scale up to 13–16px. Color: currentColor for state inheritance, or --blue directly in decorative contexts.
